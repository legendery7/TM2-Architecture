# Future Roadmap

Planned evolution of the platform (design direction, not delivery promises).

## Near term

- Harden warehouse / worker empty-state UX
- Clarify HQ upgrade levels (schemas + menu slots)
- Expand automated tests around placement and production idle reasons

## Mid term

- Formalize war / siege roles (`commander`, `marshal`) behind feature flags
- Optional `tm_town_progress` table for upgrades not tied only to reputation
- Stronger schema validation for building YAML packs

## Long term

- Clearer module packaging boundaries for selective deployment
- Observability hooks (metrics for production queues, claim churn, economy deltas)
- Public architecture updates when major subsystems land

Items move here only when they survive design review — not every idea becomes a roadmap line.
